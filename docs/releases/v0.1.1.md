# Release v0.1.1

Patch release focused on workflow robustness for real Windows writing sessions.

## What changed

- Accepted UTF-8 with BOM for prose files, reference lists, and manifest JSON files.
- Stripped BOM and common zero-width characters before author-year citation matching.
- Added a new `from-text` CLI command that builds the manifest and runs the Zotero workflow sequentially in one process.
- Expanded tests to cover BOM-tolerant parsing, zero-width character cleanup, and the new CLI flow.

## Why this release matters

This release fixes a real failure mode observed during manuscript preparation on Windows:

- a BOM at the start of a reference file polluted the first author token,
- which broke author-year citation matching for the first cited reference,
- and required manual cleanup before the workflow could continue.

The workflow now handles that class of input automatically.

## Validation

- `python -m pip install -e .[dev] && pytest` passed locally with 10 tests.
- End-to-end validation with BOM-encoded and zero-width-polluted prose/references passed locally through the live Zotero workflow.

## Upgrade note

If the Codex skill adapter is installed locally, reinstall it after updating the package so the skill instructions expose the new preferred `from-text` flow.
