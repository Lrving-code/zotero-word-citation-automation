# Release v0.1.2

Patch release focused on citation semantics and Word/Zotero compatibility.

## What changed

- Added helper support for simple narrative citations such as `Smith (2020)`.
- Changed grouped parenthetical citations to render as one Zotero field with multiple citation items.
- Added `suppressAuthor` handling for helper-generated narrative citation fields.
- Expanded tests to verify grouped citation rendering and narrative citation rendering at the generated `.docx` XML level.

## Why this release matters

This release fixes a real formatting problem observed during manuscript preparation:

- one bracketed citation group was previously emitted as multiple independent Zotero fields,
- which could lead to nested or duplicated parentheses after Zotero refresh,
- and made grouped citations less faithful to Zotero's expected field structure.

The workflow now emits one field per citation group and supports simple narrative forms without falling back to manual manifest edits.

## Validation

- `python -m pip install -e .[dev] && pytest` passed locally with 13 tests.
- Local XML inspection confirmed that grouped citations now emit one Zotero field per group.
- Local live workflow validation through `zotero-wordflow from-text` still passed after the parser and field-rendering changes.

## Upgrade note

If the Codex skill adapter is installed locally, reinstall it after updating the package so the skill instructions expose the `v0.1.2` narrative-citation behavior.
