# Release v0.1.3

Follow-up patch release for `v0.1.2`, focused on post-review consistency fixes.

## What changed

- Accepted the `and` form in two-author narrative citations such as `Dark and Bram (2007)`.
- Updated the Crossref user agent string to `0.1.3` so runtime metadata matches the released package version.
- Removed the README roadmap contradiction now that simple narrative citations are already supported.
- Updated the Codex installer so the legacy local skill alias is derived from the canonical skill content instead of staying as a separate hard-coded copy.

## Why this release matters

`v0.1.2` fixed the main citation-structure issues, but a follow-up review identified a few smaller inconsistencies:

- one common narrative spelling (`and`) still missed the helper,
- runtime version metadata still exposed an older version,
- and the local legacy skill alias could silently drift away from the canonical skill in later updates.

This patch closes those gaps without changing the main workflow surface.

## Validation

- `python -m pip install -e .[dev] && pytest` passed locally with 14 tests.
- Local parser validation confirmed `Dark and Bram (2007)` now resolves correctly.
- The Codex installer was rerun locally and refreshed both `zotero-wordflow` and `zotero-word-citation-automation`.

## Upgrade note

If the Codex skill adapter is installed locally, rerun the installer after updating so both the canonical skill and the compatibility alias move to `0.1.3`.
