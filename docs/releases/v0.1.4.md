# Release v0.1.4

Patch release focused on simplifying Codex skill usage and removing the legacy alias.

## What changed

- Standardized on `zotero-wordflow` as the only supported Codex skill name.
- Updated the installer to remove the retired local alias directory `zotero-word-citation-automation` during reinstall.
- Updated the Crossref user agent string to `0.1.4` so runtime metadata matches the released package version.

## Why this release matters

The workflow no longer needs two local skill names for the same functionality.

Keeping both `zotero-wordflow` and `zotero-word-citation-automation` locally increased the risk of drift and made cross-thread usage less predictable. This release simplifies the contract:

- one canonical skill name,
- one installation target,
- one stable update path.

## Validation

- `python -m pip install -e .[dev] && pytest` passed locally with 14 tests.
- The installer was rerun locally and removed the retired alias directory while refreshing `zotero-wordflow`.
- Local skill inventory confirms that only `zotero-wordflow` remains under `C:\Users\admin\.codex\skills`.
