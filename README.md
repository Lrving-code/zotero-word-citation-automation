# zotero-wordflow

`zotero-wordflow` is a Windows-first automation toolkit for turning DOI-backed references and author-year prose into a Zotero-ready Microsoft Word document.

It is designed for one very specific workflow: start from plain text plus real references, finish with a `.docx` that already contains Zotero citation fields, then let Word plus Zotero render the final in-text citations and bibliography.

## Why this project exists

Researchers often have the citation logic, the reference list, and the Word manuscript, but the handoff between them is still manual and error-prone. This project closes that gap by automating the repeatable parts while keeping Zotero and Word in the final loop.

## What it does

This project automates four connected steps:

1. verify references by DOI via Crossref,
2. import or reuse the verified references in a local Zotero collection,
3. generate a `.docx` file with `ADDIN ZOTERO_ITEM CSL_CITATION` fields already embedded,
4. let the user refresh citations and insert a bibliography in Microsoft Word through the Zotero add-in.

It also includes a helper that converts:

- natural-language prose
- a plain-text reference list

into a manifest JSON that the main workflow can run.

The workflow is tolerant of UTF-8 with BOM input files, which is common on Windows when text files are written by PowerShell or Word-adjacent tooling.

## Who this is for

- researchers writing papers in Word with Zotero
- users who need repeatable author-year citation insertion
- Windows users with local Zotero Desktop and Microsoft Word
- users who want a local, inspectable workflow instead of a black-box citation assistant

## Core principles

- DOI first: only real, machine-verifiable references should enter the pipeline
- fail loudly: ambiguous author-year matches should stop the run instead of guessing
- manifest driven: every run should be reproducible from an explicit manifest
- Word stays authoritative: the final bibliography is still generated by Zotero inside Word
- one citation group, one Zotero field: grouped citations should be emitted as a single field, not stacked field fragments

## Status

Early but usable. The current scope is intentionally conservative:

- parenthetical author-year citations such as `(Smith et al., 2020)` are supported
- narrative citations such as `Smith (2020)` are supported for helper-generated manifests
- ambiguous author-year matches fail fast
- unresolved citation groups fail fast

## Quick start

### 1. Build a manifest from prose and references

```powershell
zotero-wordflow build-manifest `
  --text examples\sample_prose.txt `
  --references examples\sample_references.txt `
  --collection-name "julei refs" `
  --output-manifest outputs\manifest.json `
  --output-dir outputs\run `
  --output-docx outputs\run\sample.docx
```

### 2. Generate the Zotero-ready Word document

```powershell
zotero-wordflow run --manifest outputs\manifest.json
```

### Preferred single-step workflow

If the starting point is plain prose plus a reference list, prefer the sequential command below. It builds the manifest and immediately runs the Zotero workflow in one process, which avoids race conditions between separate shell steps:

```powershell
zotero-wordflow from-text `
  --text examples\sample_prose.txt `
  --references examples\sample_references.txt `
  --collection-name "julei refs" `
  --output-manifest outputs\manifest.json `
  --output-dir outputs\run `
  --output-docx outputs\run\sample.docx
```

### 3. Finish inside Word

1. Open the generated `.docx`
2. Go to the `Zotero` tab
3. Set `Document Preferences`, for example `APA 7th edition`
4. Click `Refresh`
5. Put the cursor at the end of the document
6. Click `Add/Edit Bibliography`

## Example use cases

- turn a literature review section with `(Author, Year)` citations into a Word draft ready for Zotero refresh
- verify a manually compiled reference block before it enters a manuscript
- batch-generate citation-ready `.docx` drafts for iterative paper writing
- drive the workflow from Codex through the optional skill adapter

## Safety behavior

- manifest structure is validated before Zotero import starts
- relative paths in the manifest are resolved relative to the manifest file
- if document generation fails after import, the local Zotero database is restored from the pre-import backup
- exact-title reuse is disabled by default; DOI match is the default reuse path

## Limitations

- the workflow expects real DOI-backed references
- this project targets local Zotero desktop storage and Word on Windows
- the prose helper supports parenthetical groups and simple narrative forms such as `Smith (2020)`
- the final bibliography insertion still happens inside Word through the Zotero add-in
- terminal output on legacy non-UTF-8 Windows shells can still display mojibake for some Chinese paths, even though file I/O is handled in UTF-8

## Codex integration

An optional Codex skill adapter is included at:

- [integrations/codex-skill/SKILL.md](integrations/codex-skill/SKILL.md)

The canonical skill name is `zotero-wordflow`. Older local aliases are considered retired and should be removed during reinstall.

Install it on a Windows machine with:

```powershell
powershell -ExecutionPolicy Bypass -File integrations\codex-skill\scripts\install_skill_to_codex.ps1
```

The adapter expects `zotero-wordflow` to be installed in the active Python environment and forwards commands to:

```powershell
python -m zotero_wordflow ...
```
